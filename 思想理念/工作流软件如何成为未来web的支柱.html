工作流软件如何成为未来web的支柱

此文作者是 Kevin
Lindquist，工作流平台 Decisions 的营销负责人，原文发表于 VB 上。

Web 3.0 正在敲门，但是开门的人你永远都想不到：工作流软件。

传统上工作流软件是企业级的业务流程管理工具，但现在随着网络互联程度的加深，工作流软件开始呈现出其作为互联网技术的一面。

诚然，社会化媒体业通过将人连接到一起而主导了 Web 2.0，但社交网络现在也已经有 10 年历史。那么工作流软件这项比前者还要古老的企业工具怎么就会变成下一代互联网技术的平台呢？怎么就能断言 Web 2.0 来了又走了呢？且听我慢慢道来。

Web 1.0：“静态的 Web”

基本上 Web 1.0 诞生自 interwebs。回到 1990 年代，那时的 web 都是些静态的 HTML 页面，主要是通过目录和书签发现。当时互联网上的网页和信息基本上都是各自为政，就像一个个孤岛。

营销人员开始意识到和互联网广告及营销拓展的潜能，但是有助于那方面努力的软件产品却不多。

而这时候的工作流仍非常初级，主要关注于需要基本审批流程的特定应用。但是，随着人和软件在网上的对话的增多以及 web 的成熟，发生了一些有趣的事情。

Web 2.0：“社会化 Web”或“协作性 Web”

大家对什么是 Web 2.0 的争论非常激烈。有人说我们现在也还没有实现。但我认为我们不仅已经到达了 Web 2.0，甚至已经超越这个阶段了。这个时代通过新的表单、新的沟通模式为我们带来了新的价值。Web 2.0 就是让人和系统通过多个平台相互沟通的平台。

我们意识到，在网上交互方式（不仅仅是人与人之间）方面需要有某种程度的统一。我们创建了若干标准格式，使得系统之间能够快速沟通。我们通过社会化媒体实现了更好的人—人互动，通过 API 实现了更好的系统—系统交互。这是 Web 2.0 的重大事件之一。

2000 年代的这 10 年我们开发了更为直观的新办法来在网上找东西、交互，创建东西。我们有了一些基本的 API，开发者可以用来与强大的数据和结构基础集成到一起。不同的网站孤岛之间现在拉起了电话线，可通过应用、系统、平台来收发数据。我们不断地说话。说很多的话。

在 Web 1.0 时代玩转横幅广告等形式的营销专家显然很喜欢这种喋喋不休，开始想方设法插话。按点击付费，更好的内容管理系统，搜索引擎优化软件，客户关系管理软件，社会化媒体管理工具，等等，为了管理不断飙涨的网上流量，对各种各样的营销软件的需求也剧增。

工作流这段时间也没闲着。通过工作流平台生成和管理自动化流程，大型企业 IT 系统的管理也变得更容易了。排名算法和推荐以及自定义组件这些工作流平台已经有了一段时间的东西也冒出来了，但是这些技术至今仍有待应用。

欢迎来到 Web 3.0：“一体化 Web”（不是“语义 Web”）

许多人正在努力推进所谓的“语义 Web”，即应用可以以这样的方式理解用户交互，它不仅可以返回直接相关结果，而且还能返回间接相关结果。这种概念是令人兴奋，具有前瞻性和革新性，但是，在社会化 Web 和语义 Web 之间仍少了关键的一个演进步骤：即能取出一个对象，然后通过多项服务传递过去来完成任务。

让应用返回间接相关结果有点难，但是让应用在得到允许的情况下自动执行哪些间接相关服务如何呢？换句话说，如果你要给我展现“语义 Web”，那就展现这样一个语义 Web 吧—在这样的 Web 上，应该不需要我在不同的应用、网页等之间来回穿梭就能把事情干完，行不？

比方说，现在有一个对象，一位客户，该客户需要被引导到越来越多不关联的营销及销售工具去，最后返回完成任意数量的指定任务。营销需要 web 分析数据进行归因分析，然后以表单形式提交给 CMS 进行处理。电子邮件广告也需要内置到故事当中以便我们了解多少接触点才能转化为广告点击。哦，顺便说一下，销售用软件对电话进行记录并管理热线，能不能帮忙问问看看点击了广告的人访问了哪些内容，或者哪些他们回了哪些电子邮件？然后我们还有计费流程和记账软件等等一堆待处理的东西。

工作流软件就是这一切事务的粘合剂，是 Web 3.0 中把一切聚拢到一起的基础。“社会化”把人联系到一起，而“工作流”则把公司、系统、技术、应用和人连接起来。这里要解决的不仅仅是 CRM 问题。还要解决大数据问题，云安全问题等等。

拥抱 Web 3.0

要想利用好 Web 3.0 的演进，你需要跨服务管理好你的流程。随着时间的推移，集成多种服务的产品会越来越多。营销自动化仅仅是开始。然而，指望这些服务演进为完整一体化的服务解决方案是不现实的。

好的工作流平台会迎合特定需求和技术栈，但实现起来要费点功夫。伟大的工作流平台会把工作流的主要组件（表单、工作流、规则、仪表盘）打包进一个个的服务包中，让你可以把一个个点连接起来，然后开始把流程跑起来。

我很高兴看到大家用工作流软件来创建 Web 3.0 服务，这不仅能让语义 Web 成为现实，而且是非常有意义的现实。
