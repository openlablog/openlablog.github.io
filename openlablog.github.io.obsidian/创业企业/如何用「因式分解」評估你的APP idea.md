最近感覺到，並不是執行比 idea 重要，而是 idea 和執行一樣重要，甚至 idea 還比執行重要一些些。

在有這個想法後，小黑的這一篇 [好文](http://startup.epromotor.info/startup/ec-win-win-formula-organic-traffic/)，讓我思考起，如何用因式分解來評估點子這件事，可以稍微做到不偏頗的來看待 idea 的評估。

先來看一下小黑提的，電子商務營業額的因式分解：

{（廣告投入金額 / 單次點擊費用）＋（導入自媒體流量 X 媒介轉換率）＋[（用戶需求關鍵字搜索量 X 搜索引擎優化率）＋（（滿意消費者文章分享數 + 電子媒體報導該篇瀏覽數）X 內容有帶上網址點擊比例）＋（（受滿意消費者影響用戶數 + 受未帶網址電子媒體報導影響用戶數 + 受線下媒體影響用戶數）X 實際搜尋比例）+ 直接流量 + 舊顧客回訪數] } X 提袋率 X 客單價 = 營業額

這樣的一個因式分解，啟發了我也想如法炮製一個關於 APP 的因式分解：

- 營業額= 活躍使用者 x 商業模式
- 營業額= 下載數 x 回訪率 x 商業模式
- 營業額= 下載數 x (需求強度 x 需求頻率 x 回訪轉換率) x 商業模式
- 營業額= [Mobile AD + 自有媒體 + Viral + 自然下載 + 媒體報導 + (排行榜外部性 x 評價)] x (需求強度 x 需求頻率 x 回訪轉換率) x 商業模式
- 營業額= [Mobile AD + (SEO + Social + Direct 流量) + (Viral 因子 x Viral 種子)+ 自然下載 + 媒體報導 + (排行榜外部性 x 評價)] x (需求強度 x 需求頻率 x 回訪轉換率) x 商業模式
- 營業額= {Mobile AD + [(用戶需求關鍵字搜索量 x 搜索引擎優化率) + (議題操作擴散度 x 議題轉換率) + Direct 流量] + (Viral 因子 x Viral 種子) + 自然下載 + 媒體報導 + (排行榜外部性 x APP 使用者經驗 x 鼓勵評價機制)} x (需求強度 x 需求頻率 x 回訪轉換率) x 商業模式

這樣拆解有個好處，可以很簡單的區分點子面和執行面。

![](assets/如何用「因式分解」評估你的APP%20idea/file-20.4629.jpg)

APP idea 會影響到你的成長天花板和成長的斜率，在實作你的 APP 之前，就應該要評估這六個面向。idea 評估完畢之後，要實作出來，除了 APP 開發的能力外，也該看看團隊是否擁有執行該有的九個能力。

至於商業模式的部分，除了要考慮關於 [成長的issue](http://mrpm.cc/?p=817)，也要考慮如何簡化「把流量換錢的方法」，這部份的感想改天再來寫一篇好了。
